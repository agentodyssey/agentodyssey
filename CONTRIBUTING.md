## Reporting Bugs

- Before opening a new bug report, please search existing issues and the [troubleshooting guide](http://agentodyssey.github.io/docs/troubleshooting) to see whether it has already been reported.
- If the issue is new, open an issue and include as much detail as possible.

```markdown
**Description**
A clear and concise description of what the bug is.

**To Reproduce**
Example code or commands to reproduce the bug.

**Expected Behavior**
A clear and concise description of what you expected to happen.

**Screenshots / Logs**
If applicable, add screenshots or logs to help explain the problem.

**Environment**
- OS: [e.g., Linux, macOS]
- Python version: [e.g., 3.10, 3.11, 3.12]
- GPU/CPU: [e.g., A100, RTX 4090, M3 Pro]

**Additional Context**
Add any other context about the problem here.
```

## Suggesting Features

- If you have a feature request, please open an issue labeled enhancement.
- Describe the motivation, expected behavior, and any potential trade-offs.
- If possible, include example commands or API usage to show the desired workflow.

## Submitting Code Changes

- Fork the repository and create a focused feature branch.
- Keep pull requests small and scoped to one change when possible.
- Add or update documentation when behavior, CLI arguments, or public APIs change.
- Before opening a PR, run relevant checks locally (for example, quick generation/evaluation flows you modified).
- In your pull request title, include one of: [BUG FIX], [FEATURE], or [MISC].
- In the PR description, include:
  - What changed
  - Why it changed
  - How you tested it (commands and expected outcomes)

## Reviewing and Merging

- PRs require at least one approval before merging.
- Prefer Squash and Merge to keep commit history clean and readable.

## Questions and Discussions

- For open-ended questions and design discussions, start a GitHub issue or discussion thread.
- Use issues for actionable work items and reproducible bugs.
